Django报错记录

发表于 2020-12-11 更新于 2020-12-14 分类于 Python

使用source 命令时提示命令不存在

需要source 的文件中source拼写错误

post请求时403错误

  1. 在settings.py里面的MIDDLEWARE_CLASSES中加入“‘django.middleware.csrf.CsrfResponseMiddleware’,”；

  2. 在settings.py里面的MIDDLEWARE_CLASSES中注释掉“‘django.middleware.csrf.CsrfViewMiddleware’,”。

使用ajax时报错:TypeError: $.ajax(…) success is not a function

success放到ajax里面

<script>
$.ajax({
    type: "GET",
    url: "/api/rooms",
    success: function (rooms) {
    }
});
</script>

在视图中使用模型查询时报错:’str’ object has no attribute ‘objects’

在模型中定义类时,类名要大写

使用ajax时JsonResponse直接返回页面,没有回调

使用ajax时不能和input的submit一起使用

Django无法导入jQuery报错:the server responded with a status of 404

在setting文件中添加

STASTICFILES_DIRS = [os.path.join(BASE_DIR, 'stastic')] #'stastic'为在项目目录下创建的静态文件夹

在a标签中提交表单

<form action="" id='form1'>...</form>

<a href='javascript:;' onclice='document:form1.submit()'>...</a>

views中函数取cookie时键名不存在报错

先检查键名是否存在与cookie中,若存在再取

if 键名 in request.COOKIES:

    pass

jquery获取标签内的内容

  1. 获取标签的html文件

    $(“#treeDemo”).html()

  2. 2.获取标签内的文本内容

    $(“#treeDemo”).text()

  3. 3.清空标签下的内容

    $(“#treeDemo”).empty()

视图函数给模板传参数时报错:dictionary update sequence element #0 has length 1; 2 is required

传递的参数需要以字典的形式传送
