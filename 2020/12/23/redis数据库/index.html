redis数据库

发表于 2020-12-23 更新于 2020-12-26

nosql介绍

NoSQL:一类新出现的数据库(not only sql),它的特点:

  • 不支持SQL语法

  • 存储结构跟传统关系型数据库中的那种关系表完全不同,nosql中存储的数据都是KV形式

  • NoSQL的世界中没有一种通用的语言,每种nosql数据库都有自己的api和语法,以及擅长的业务场景

  • NoSQL中的产品种类相对多:

    1. Mongodb

    2. Redis

    3. Hbase hadoop

    4. Cassandra hadoop

NoSQL和SQL数据库的对比

  • 适用场景不同:sql数据库适用于关系特别复杂的数据查询场景,nosql则相反

  • 事务特性的支持:sql对事务的支持非常完善,而nosql基本不支持事务

  • 两者在不断地取长补短,呈现融合趋势

Redis简介

  • Redis是一个开源的使用ANSIC语言编写,支持网络,可基于内存亦可持久化的日志型,Key-Value数据库,并提供多种语言的API.

  • Redis是NoSQL技术阵营中的一员,它通过多种键值数据类型来适应不同场景下的需求,借助一些高级层的接口使用其可以胜任,如缓存,队列系统的不同角色

Redis应用场景

  • 用来做缓存–Redis的所有数据是放在内存中的(内存数据库)

  • 可以在某些特定应用场景下代替传统数据库–比如社交类应用

  • 在一些大型系统中,巧妙地实现一些特定功能–session共享,购物车

安装

  • 下载Redis压缩包

    wget http://download.redis.io/releases/redis-4.0.9.tar.gz

  • 解压

    tar -zxvf redis-4.0.9.tar.gz

  • 复制,放到usr/local目录下

    sudo mv ./redis-4.0.9 /usr/local/redis/

  • 进入redis目录

    cd /usr/local/redis

  • 生成

    sudo make

  • 测试

    sudo make test

  • 安装,将Redis命令安装到/usr/local/bin

    sudo make install

  • 安装完成后,进入/usr/local/bin查看

    cd /usr/local/bin

ls -a

    1. redis-server redis服务器

    2. redis-cli redis命令行客户端

    3. redis-benchrmark redis性能测试工具

    4. redis-check-aof AOF文件测试工具

    5. redis-check-rdb RDB文件检索工具

  • 配置文件,移动到/etc/目录下

    sudo cp /usr/local/redis/redis.conf /etc/redis/redis.conf

配置

redis的配置信息在/etc/redis/redis.conf

核心配置选项

  • 绑定ip: 如果需要远程访问,可将此行注释,或绑定一个真实ip

    bind 127.0.0.1

  • 端口:默认为6379

    port 6379

  • 是否以守护进程运行

    daemonize yes

    1. 如果以守护进程运行,则不会在命令行堵塞,类似于服务

    2. 如果以非守护进程运行,则当前终端被堵塞

    3. 设置为yes表示守护进程,设置为no表示非守护进程

    4. 推荐设置为yes

  • 数据文件

    dbfilename dump.rdb

  • 数据文件存储路径

    dir /var/lib/redis

  • 日志文件

    logfile /var/log/redis/redis-server.log

  • 数据库,默认有16个

    database 16

  • 主从复制,类似于双机备份

    slaveof

服务器端和客户端命令

服务器端

  • ps aux | grep redis 查看redis服务器进程

  • sudo kill -9 pid 杀死redis服务器

  • sudo redis-server /etc/redis/redis.conf 指定加载的配置文件

客户端

  • redis-cli 连接redis

  • ping 运行测试

  • select n 切换数据库

数据操作

数据结构

  • redis是key-value的数据结构,每条数据都是一个键值对

  • 键的类型是字符串

  • 键不能重复

  • 值的类型分为五种

    1. 字符串string

    2. 哈希hash

    3. 列表list

    4. 集合set

    5. 有序集合zset

string

保存

如果设置的键不存在则为添加,如果设置的键已经存在则修改

  • 设置键值

    set key value

  • 例1 设置键为name值为itcast的数据

    set name itcast

get name

set name iteima

get name

  • 设置键值及过期时间,以秒为单位

    setex key seconds value

  • 例2 设置键为aa值为aa过期时间为3秒的数据

  • 设置多个键值

    mset key1 value1 key2 value2

  • 追加值

    append key value

获取

  • 根据键获取值,如果不存在此键则返回nil

    get key

  • 根据多个键获取多个值

    mget key1 key2

键命令

  • 查找键,参数支持正则表达式

    keys pattem

  • 例1: 查看所有键

    keys *

  • 例2: 查看名称中包含a的键

    keys 'a*'

  • 判断键是否存在,如果存在范湖1,不存在返回0

    exsits key1

  • 查看键对应的value的类型

    type key

  • 删除键及对应的值

    del key1 key2

  • 设置过期时间,以秒为单位

    expire key seconds

  • 查看有效时间,以秒为单位

    ttl key

hash

  • 用于存储对象,对象结构为属性,值

  • 值为类型为string

增加修改

  • 设置单个属性

    hset key field value

  • 例1: 设置键user的属性name为itheima

    hset user name itheima

  • 设置多个属性

    hmset key field1 value 1 field2 value2

获取

  • 获取指定键所有的属性

    hkeys key

  • 获取一个属性的值

    hget key field

  • 获取多个属性的值

    hmget key field1 field2

  • 获取所有属性的值

    hvals key

删除

  • 删除整个hash键及值,使用del

  • 删除属性,属性对应的值会被一起删除

    hdel key field1 field2

list

  • 列表的元素类型为string

  • 按照插入顺序排序

增加

  • 在左侧插入数据

    lpush key value1 value2

  • 在右侧插入数据

    rpush key value1 value2

  • 在指定元素的前后插入新元素

    linsert key before 或 after 指定元素 新元素

获取

  • 返回列表里指定范围内的元素

    1. start,stop 为元素下标索引

    2. 索引从左侧开始,第一个元素为0

    3. 索引可以是负数,表示从尾部开始计算,如-1表示最后一个元素

      lrange key start stop

设置指定索引位置的元素值

lset key index value

删除

  • 删除指定元素

    1. 将列表中钱count次出现的值为value的元素移除

    2. count > 0:从头往后移除

    3. count < 0:从尾往头移除

    4. count == 0:移除所有

      lrem key key count value

set

  • 无序集合

  • 元素为string类型

  • 元素具有唯一性,不重复

  • 对于集合没有修改操作

增加

  • 添加元素

    sadd key key member1 member2

获取

  • 获取所有元素

    smembers key

删除

  • 删除指定元素

    srem key value1 value2

zset

  • sorted set,有序集合

  • 元素为string类型

  • 元素具有唯一性,不重复

  • 每个元素都会关联一个double类型的score,表示权重,通过权重将元素从小到大排序

  • 没有修改操作

增加

  • 添加

    zadd key score1 member1 score2 member2

获取

  • 获取所有元素

    zrange key start stop

  • 获取score值在min和max之间的元素

    zrangebyscore key min max

  • 获取成员member的score值

    zscore key member

删除

  • 删除指定元素

    zrem key member1 member2

  • 删除权重在min和max之前的元素

    zremrangebyscore key min max

Django

Django存储session

  • 之前的Django的session默认是存在数据库里的,我们也可以把session存储在Redis里面

准备工作

  • 创建Django项目和应用

  • 配置url

session的redis存储配置

  • 安装包

    pip3 install django-redis-sessions

  • 在setting.py中配置使用redis

    SESSION_ENGINE = 'redis_sessions.session'

SESSION_REDIS_HOST = 'localhost'

SESSION_REDIS_PORT = 6379

SESSION_REDIS_DB = 0

SESSION_REDIS_PASSWORD = ''

SESSION_REDIS_PREFIX = 'session'
